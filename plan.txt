Этап 0: Подготовка (1-2 дня)
1. Создать структуру проекта на GitHub/GitLab
2. Настроить локальное окружение (Go, Docker, Git)
3. Создать README.md с описанием проекта
4. Составить детальный Task Breakdown

Этап 1: Проектирование (2-3 дня)
1. Детальное проектирование БД (таблицы, связи, индексы)
2. Проектирование API (полная спецификация ручек)
3. Проектирование структуры кода Go
4. Создание диаграмм архитектуры

Этап 2: Базовая инфраструктура (2 дня)
1. Написать Dockerfile для Go-сервиса
2. Создать docker-compose.yml (Go + PostgreSQL)
3. Настроить миграции БД
4. Настроить подключение к БД из Go

Этап 3: MVP по людям (3-4 дня)
1. Реализовать CRUD для людей (/api/people)
2. Простой интерфейс для тестирования (можно Swagger или простой HTML)
3. Базовые тесты API

Этап 4: Связи между людьми (3-4 дня)
1. Реализовать CRUD для связей
2. Валидация связей (нельзя создать циклические связи)
3. Обновить тестовый интерфейс

Этап 5: Алгоритм дерева (3-5 дней)
1. Реализовать алгоритм построения дерева
2. Эндпоинт для получения дерева
3. Тесты на разные семейные структуры

Этап 6: Поиск и интерфейсы (4-6 дней)
1. Реализовать поиск по людям
2. Создать базовый административный интерфейс
3. Создать базовый пользовательский интерфейс

Этап 7: Доработки (3-5 дней)
1. Загрузка фото
2. Валидация данных
3. Оптимизация запросов
4. Документация API
